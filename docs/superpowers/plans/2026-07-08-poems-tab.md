# Şiirlerim Sekmesi Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bloga mevcut kategori mimarisine uyan yeni bir "Şiirlerim" (`poems`) kategorisi eklemek: sidebar sekmesi, liste sayfası ve şiir detay sayfaları.

**Architecture:** Site tamamen konfigürasyon üzerinden kategori üretiyor: `content.config.ts` koleksiyonu tanımlar, `configuration.ts`'teki `ENTRY_CATEGORIES`/`NAVIGATIONS`/`CATEGORIES_LABELS` rota ve sidebar sekmesini üretir. `Category` tipi `CATEGORIES_LABELS`'tan türediği için tip değişikliği gerekmez. Yeni kod yazılmaz; iki dosyaya konfigürasyon eklenir ve örnek içerik konur.

**Tech Stack:** Astro 5 (content collections, glob loader), TypeScript, Vercel adapter.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-08-poems-tab-design.md`
- Sidebar etiketi: `Şiirlerim` (tr) / `Poems` (en), URL: `/poems`.
- Şema: mevcut `entrySchema` aynen kullanılır; yeni şema tanımlanmaz.
- Şiir içeriği yalnızca `tr.mdx` olarak eklenir (devam eden tek-dilli geçişle uyumlu).
- Test framework yok; doğrulama `npm run build` ve build çıktısı kontrolüyle yapılır.
- Çalışma ağacında bu işle ilgisiz i18n-kaldırma değişiklikleri var: `git add` HER ZAMAN dosya yoluyla yapılır, asla `git add -A`/`git add .` kullanılmaz.

### Task 1: `poems` koleksiyonu ve örnek şiir içeriği

**Files:**
- Modify: `src/content.config.ts` (cultures tanımının sonrası, satır ~59 ve collections export'u satır 61-66)
- Create: `src/content/blog/poems/ilk-siirim/tr.mdx`

**Interfaces:**
- Consumes: `content.config.ts` içindeki mevcut `entrySchema` fonksiyonu (satır 4-13).
- Produces: `poems` adında bir Astro content collection. Task 2'deki `getEntries({ category: "poems", locale })` çağrısı (mevcut util, `src/utils/request.ts:3`) bu koleksiyon adına dayanır. Entry id formatı `<slug>/tr` (klasör/dosya yapısından gelir).

- [ ] **Step 1: `src/content.config.ts`'e poems koleksiyonunu ekle**

`cultures` tanımının (satır 53-59) hemen altına ekle:

```ts
const poems = defineCollection({
  loader: glob({
    pattern: "**/[^_]*.mdx",
    base: "./src/content/blog/poems",
  }),
  schema: entrySchema,
});
```

`collections` export'unu şöyle güncelle:

```ts
export const collections = {
  journey,
  writings,
  thoughts,
  cultures,
  poems,
};
```

- [ ] **Step 2: Örnek şiir dosyasını oluştur**

`src/content/blog/poems/ilk-siirim/tr.mdx` dosyasını şu içerikle oluştur:

```mdx
---
title: İlk Şiirim
description: Şiirler bölümünün ilk örneği.
date: 2026-07-08
topic: şiir
tags: ["şiir"]
---

Bu bir örnek şiirdir,
Yerine gerçeği gelecektir.

Satırlar kısa, anlam derin,
Bu dosya sadece bir denemedir.
```

- [ ] **Step 3: Build ile koleksiyonun yüklendiğini doğrula**

Run: `npm run build`
Expected: Build hatasız tamamlanır (exit code 0). Bu aşamada henüz poems sayfası üretilmez (rota konfigürasyonu Task 2'de); doğrulanan şey koleksiyon tanımının ve frontmatter şemasının geçerli olduğudur. Şema hatası olsaydı build `[InvalidContentEntryDataError]` ile düşerdi.

- [ ] **Step 4: Commit**

```bash
git add src/content.config.ts src/content/blog/poems/ilk-siirim/tr.mdx
git commit -m "feat: add poems content collection with sample poem"
```

### Task 2: Konfigürasyona poems kategorisini ekle (sekme + rotalar)

**Files:**
- Modify: `src/configuration.ts` (`ENTRY_CATEGORIES` ~satır 25-29, `NAVIGATIONS` ~satır 30-56, `CATEGORIES_LABELS` ~satır 57-78)

**Interfaces:**
- Consumes: Task 1'in ürettiği `poems` koleksiyonu.
- Produces: `C.ENTRY_CATEGORIES` içinde `"poems"` (rota üretimi `src/pages/[lang]/[category]/index.astro:8` bunu okur), `C.CATEGORIES_LABELS.poems` (`Category` tipi ve sayfa başlıkları bunu okur), `C.NAVIGATIONS` içinde poems girdisi (sidebar sekmesi bunu okur).

- [ ] **Step 1: `src/configuration.ts`'i güncelle**

`ENTRY_CATEGORIES` dizisine `"thoughts"`ın altına ekle:

```ts
  ENTRY_CATEGORIES: [
    "writings",
    "cultures",
    "thoughts",
    "poems",
  ],
```

`NAVIGATIONS` dizisine `thoughts` girdisinden sonra ekle:

```ts
    {
      name: "poems",
      href: "/poems",
      locales: ["tr"],
    },
```

`CATEGORIES_LABELS`'a `thoughts` girdisinden sonra ekle:

```ts
    poems: {
      tr: "Şiirlerim",
      en: "Poems",
    },
```

Not: Çalışma ağacındaki i18n-kaldırma değişiklikleri bu dosyanın yapısını sadeleştirmiş olabilir (ör. `CATEGORIES_LABELS` düz string map'e inmiş, `NAVIGATIONS`'dan `locales` kalkmış olabilir). Dosyanın o anki halindeki mevcut kategori girdilerinin (writings/thoughts) kalıbını birebir kopyala; alan adları yukarıdakinden farklıysa dosyadaki kalıp esastır.
