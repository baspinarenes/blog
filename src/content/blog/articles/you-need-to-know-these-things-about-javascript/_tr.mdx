Javascript ise bu standartların uygulamasıdır.

2008'den önce Javascript sadece tarayıcıda çalışıyordu ancak bu Javascript'in bir sınırlaması değil kodun nerede execute edildiğine ilişkin bir kısıtlamaydı.

Bir JavaScript uygulaması başlatıldığında, bu uygulamadaki tüm kod belleğe yüklenir. 

Some of these functions come in the form of event listeners—function objects that run a corresponding callback function when an event with a matching name is emitted. These functions sit around in memory until event emitters—objects that fire event names—trigger the event listeners to run their callback functions.

Bu şekilde Node.js uygulamaları özellikle hızlı ve verimli bir şekilde çalıştırabilir. Diğer platformlar, belirli bir komutu çalıştırmak için her istek yapıldığında tüm kodlarını yeniden derlemek veya çalıştırmak zorunda kalabilirken, Node.js JavaScript kodunu yalnızca bir kez yükler ve işlevleri ve ilgili geri arama işlevlerini yalnızca olaylar tarafından tetiklendiğinde çalıştırır.

Event-driven development -- Node.js JavaScript kodunu yalnızca bir kez yükler ve işlevleri ve ilgili geri arama işlevlerini yalnızca olaylar tarafından tetiklendiğinde çalıştırır. JavaScript bir dil olarak olay güdümlü geliştirmeyi destekler ancak bunu gerektirmez. Node.js, Node.js event-loop (bkz. ünite 1) kullanarak sunucunun bir uygulamanın görevlerinin çoğunu yürütmesi için bir yol olarak olayların kullanımını teşvik ederek bu mimariden yararlanır.

Asyncronous -- Node.js yalnızca tek bir iş parçacığı gerektirir (event-loop tarafından kullanılır) ve bu iş parçacığı diğer iş parçacıklarını yalnızca daha büyük görevler için gerektiğinde kullanabilir.
