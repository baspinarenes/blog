**Single Level of Abstraction (SLA)**, fonksiyondaki ifadelerin aynı soyutlama düzeyinde (**abstraction level**) olması gerektiğini belirten ve fonksiyonların daha okunabilir olmasını amaçlayan bir programlama prensibidir. Fonksiyonları atomlarına parçalamanızı sağlar.

<MessageBox type="info">
Konuyla ilgili diğer makalelerde, soyutlama düzeyinin ne olduğunu açıklamadan (adeta sahneleri kesilmiş korsan film edasıyla) örneklere atlıyorlar. Neyse ki masadayız ve en önemli noktaya değinmeden geçmeyeceğiz.
</MessageBox>

## Soyutlama Düzeyi Nedir?

Her fonksiyon bir eylem gerçekleştirir. Bu eylemler de çoğu zaman alt eylem içerir. Bu alt eylemlerin seviyesi, soyutlama seviyesini belirtir. Esasen olan budur.

Kahve makinesini örneği üzerinden ele alalım.

Bir makineden kahve almak istersek parayı atar, istediğimiz çeşidi seçer, hazırlanmasını bekler ve alırız. Bu adımlar, birinci soyutlama düzeyindedirler çünkü otomattan kahve almak için gereken temel adımlardır.

Kullanıcının etkileşime girdiği bu eylemler, arkaplanda başka eylemleri harekete geçirir. Kahve hazırlama aşamasında makine; bardağı alır, kahveyi koyar, sıcak su ekler ve karıştırır. Bu adımlar ise ikinci soyutlama düzeyindedirler (parayı alma eyleminin alt eylemleri de aynı şekildedir). Kahve almak isteyen kişi bardağın nasıl geldiğini ya da kahvenin nasıl karıştırıldığını görmesi gerekmez. Ancak makine bardak koymuyorsa o zaman bardağı koyan mekanizmayı kontrol edebilir.

Fonksiyonlar da böyledir. Single Level of Abstraction prensibini ihlal eden basitleştirilmiş kod aşağıdaki gibidir.

```java
getCoffee() {
    checkMoney();       // abstraction level 2
    addToBallance();    // abstraction level 2
    chooseCoffeeType(); // abstraction level 1
    getGlass();         // abstraction level 2
    putCoffee();        // abstraction level 2
    putWater();         // abstraction level 2
    shuffleGlass();     // abstraction level 2
    giveCoffee();       // abstraction level 1
}
```

Kodu ekstra soyutlaştırma ekleyecek şekilde refactor etmemiz gerekir ve son kod aşağıdaki gibi olur.

```java
getCoffee() {
    getMoney();         // abstraction level 1
    chooseCoffeeType(); // abstraction level 1
    prepareCoffee();    // abstraction level 1
    giveCoffee();       // abstraction level 1
}
```

## Kod Örnekleri

`calculateSum` fonksiyonu SLA prensibine uygundur ve aynı soyutlama düzeyindeki ifadeleri içerir. Uygun olmasının sebebi, yalnızca toplamın hesaplanmasıyla ilgili ayrıntıları içermesidir. Hesaplama sonucunu tutan bir değişken tanımlarız, diğer sayıları buna ekleriz ve döndürürüz.


```java showLineNumbers
public int calculateSum(int[] numbers) {
  int sum = 0;

  for (int num : numbers) {
   sum += num;
  }

  return sum;
}
```

`validateUser()` fonksiyonu ise farklı soyutlama seviyeleri içerir. `isValidEmail` ikinci soyutlama seviyesindedir. Fonksiyonun asıl amacı kullanıcının geçerli bir kullanıcı olduğunu kontrol etmektir. Ancak koda baktığımızda, doğrulama ve loglama işlemlerinin detaylarını da görürüz. Bu da birden fazla soyutlama düzeyinin varlığına işaret eder. Okunabilirliklerini karşılaştırabilmeniz adına `validateUser()` fonksiyonunu biraz inceleyin.

```java
public boolean validateUser(User user) {
  if (user.getName().isEmpty()) {
    System.out.println("Name cannot be empty.");
    return false;
  }

  if (user.getEmail().isEmpty()) {
    System.out.println("Email cannot be empty.");
    return false;
  }

  if (!isValidEmail(user.getEmail())) {
    System.out.println("Invalid email format.");
    return false;
  }

  if (user.getPassword().isEmpty()) {
    System.out.println("Password cannot be empty.");
    return false;
  }

  if (user.getPassword().length() < 8) {
    System.out.println("Password must be at least 8 characters long.");
    return false;
  }

  return true;
}
```

`validateUser`'ın SLA prensibine uygun hali ise aşağıdaki gibidir.

```java
public boolean validateUser(User user) {
  if (!hasValidName(user)) { // isim doğrulama detayları soyutlandı
    logError("Invalid name"); // loglama detayları soyutlandı
    return false;
  }

  if (!hasValidEmail(user)) { // email doğrulama detayları soyutlandı
    logError("Invalid email"); // loglama detayları soyutlandı
    return false;
  }

  if (!hasValidPassword(user)) { // şifre doğrulama detayları soyutlandı
    logError("Invalid password"); // loglama detayları soyutlandı
    return false;
  }

  return true
}
```

Yeni fonksiyonlarla birlikte kod miktarındaki artışa rağmen ne kadar kolay okunabildiğine dikkat edin. Bu kodla ilgilenen yazılımcı olduğunuzu ve yaş doğrulamasının olup olmadığını kontrol ettiğinizi düşünün:

- `validateUser` fonksiyonunu kontrol edersiniz.
- Hangi doğrulama aşamalarını barındırdığını hızlıca görüp yaş doğrulamasının olmadığını farkedersiniz.
- Gerekliyse yaş doğrulaması için doğrulama fonksiyonu oluşturup koda entegre edersiniz.

Bitti gitti. Bunu kontrol ederken şifre ya da email doğrulama detayları umrunuzda değildir.

## Uygulama

Verdiğimiz ürün listesinin toplam fiyatını hesaplayan bir fonksiyonunuzun olduğunu varsayın ve bir dakikanızı ayırarak kodun SLA prensibini neden ihlal ettiğini analiz edin. **Emeksiz Ekmek Olmaz (EEO)** prensibi gereği kendiniz refactor etmeye çalışın.

```java
public int calculateTotalCartPrice(CartItem[] cartItems) {
  int totalPrice = 0;

  for (CartItem cartItem : cartItems) {
    totalPrice += cartItem.getPrice();

    if (cartItem.isTaxable()) {
      totalPrice += cartItem.getPrice() * 0.18;
    }

    if (cartItem.getCategory().equals("Electronics")) {
      if (cartItem.getPrice() > 500) {
        totalPrice -= 50;
      }
    }
  }

  return totalPrice;
}
```

Fonksiyonu incelediğimizde ürün fiyatın toplama eklediğini, üzerine ürün vergisini eklediğini ve son olarak mevcutsa indirimi düştüğünü görebiliriz. Kodumuz üç parçadan oluşuyor. Adım adım refactor edelim:

- `calculateTotalCartPrice`'daki döngü içinde ürün fiyatını hesaplayan mantık kodun geri kalanından farklı soyutlama seviyesindedir. Ayrı bir fonksiyona çıkaralım.

<MessageBox type="info">
Bir fonksiyonda döngü varsa Single Level of Abstraction'a aykırı kod parçası içermesi muhtemeldir.
</MessageBox>

```java
public int calculateTotalCartPrice(CartItem[] cartItems) {
  int totalPrice = 0;

  for (CartItem cartItem : cartItems) {
    totalPrice += calculateCartItemPrice(cartItem);
  }

  return totalPrice;
}

private int calculateCartItemPrice(CartItem cartItem) {
  int cartItemPrice = cartItem.getPrice();       // abstraction level 1

  if (cartItem.isTaxable()) {
    cartItemPrice += cartItem.getPrice() * 0.18; // abstraction level 2
  }

  if (cartItem.getCategory().equals("Electronics")) { 
    if (cartItem.getPrice() > 500) {             // abstraction level 2
      cartItemPrice -= 50;
    }
  }

  return cartItemPrice;
}
```

- `calculateTotalCartPrice` fonksiyonunu başarıyla refactor ettik ancak yeni fonksiyonumuz da farklı soyutlama düzeylerine sahiptir. Bu koda baktığınızda fiyatın nasıl hesaplandığını görmek istersiniz ancak verginin nasıl hesaplandığını veya indirimin nasıl belirlendiğini görmek istemezsiniz.

```java
public int calculateTotalCartPrice(CartItem[] cartItems) {
  int totalPrice = 0;

  for (CartItem cartItem : cartItems) {
    totalPrice += calculateCartItemPrice(cartItem);
  }

  return totalPrice;
}

private int calculateCartItemPrice(CartItem cartItem) {
  int cartItemPrice = cartItem.getPrice();

  if (cartItem.isTaxable()) {
    cartItemPrice += calculateTax(cartItem.getPrice());
  }

  if (isElectronicItem(cartItem)) {
    cartItemPrice -= calculateDiscount(cartItem.getPrice());
  }

  return cartItemPrice;
}

private int isElectronicItem(CartItem cartItem) {
  return cartItem.getCategory().equals("Electronics");
}

private int calculateTax(int price) {
  return price * 0.18;
}

private int calculateDiscount(int price) {
  return price > 500 ? 50 : 0;
}
```

Refactor sonucunda pirüpak ve okunaklı bir kod elde etmiş oluruz.
